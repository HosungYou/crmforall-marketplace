---

크렘포올 커넥터 MCP를 사용해 온보딩을 순서대로 진행해줘.

**중요 — 결정 지점에서는 반드시 `AskUserQuestion` 도구로 선택지를 제시한다**(자유 서술로 "어떻게 할까요?"
라고 묻지 말 것). 아래 4개 지점에서만 질문을 띄우고, 사용자가 고른 대로 진행한다. 그 외 단계는
결과를 1~2줄로 요약하고 다음으로 넘어간다. 비밀번호·시크릿·전화번호 원문은 절대 출력하지 않는다.

## 1. 현재 상태 확인 — `onboarding_status`

현재 단계와 다음 할 일을 확인한다.

**▶ [결정 ①] 아직 DB 연결(설정)이 없으면** `AskUserQuestion`을 띄운다:
- 질문: "아직 고객사 DB 연결이 없습니다. 어떻게 연결할까요?" / header: "연결 방법"
- 선택지:
  - **세션 안에서 바로 셋업 (권장)** — `setup_login`(브라우저 승인)→`setup_database`(입력창)로 이 세션에서 끝냄. 터미널·재시작 불필요.
  - **터미널 connect init** — 전산담당자가 분리돼 있으면 `npx @crmforall/connect init --token=<설치토큰> --platform-url=https://crmforall-console.vercel.app`. 설치토큰은 콘솔 → 운영 → 데이터 연결에서 발급.
  - **이미 연결했음 — 다시 확인** — `onboarding_status`를 다시 호출해 상태를 갱신.
- "세션 안에서…"를 고르면 `setup_login` → `setup_database`를 안내·실행한다(비밀번호는 입력창으로 직접 입력 → 모델에 노출되지 않음). 그 뒤 2단계로 진행.

## 2. 연결·권한 점검 — `test_connection`

연결과 권한을 점검한다.

**▶ [결정 ②] 과권한(원본 테이블 INSERT/UPDATE/DELETE 권한)이 감지되면** `AskUserQuestion`을 띄운다:
- 질문: "원본 테이블 쓰기 권한이 감지돼 보안상 활성화가 차단됩니다. 어떻게 할까요?" / header: "과권한 대응"
- 선택지:
  - **권한 축소 SQL 받기 (권장)** — 출력된 REVOKE SQL을 그대로 DBA에게 전달하고, 적용 후 `test_connection` 재시도.
  - **읽기 전용 계정으로 재연결** — `setup_database`를 다시 호출해 최소권한(SELECT만 + crmforall 스키마 CREATE) 계정으로 연결.
  - **지금 중단** — 여기서 멈추고 사용자가 DBA와 협의하도록 안내.
- 통과하면 3단계로 진행.

## 3. 스키마 탐색 — `inspect_schema`

테이블·컬럼을 살펴 핵심 후보(고객/회원, 예약/주문 등)를 1~2줄로 요약한다. (질문 없음)

## 4. 필드 매핑 — `propose_field_mapping` (핵심 결정)

표준 필드 매핑 후보(고객·예약)를 만들고, 사람이 읽기 쉽게 표로 요약한다(표준필드 → 감지된 컬럼).

**▶ [결정 ③] 매핑 요약 직후 `AskUserQuestion`을 띄운다:**
- 질문: "제안된 표준 필드 매핑을 확정할까요?" / header: "매핑 확정"
- 선택지:
  - **제안대로 확정 (권장)** — 이대로 `validate_mapping`을 persist=true로 호출해 확정.
  - **일부 필드 수정 후 확정** — 어떤 표준필드의 컬럼을 바꿀지 사용자에게 물어 조정한 뒤 확정.
  - **컬럼별 상세 먼저 보기** — 매핑을 컬럼 단위로 펼쳐 보여주고 이 질문을 다시 띄운다.
- 확정이면 `validate_mapping`(persist=true)을 호출한다. 검증 실패 시 사유를 요약하고 [결정 ③]을 다시 띄운다.

## 5. 첫 성과 + 다음 행동 — `audience_summary`

"고객 N명 · 발송 동의 M명 · 발송 가능 후보 K명"을 사람이 읽기 쉽게 보여준다(첫 성과 체감).

**▶ [결정 ④] 요약 직후 `AskUserQuestion`을 띄운다:**
- 질문: "고객 규모를 확인했습니다. 다음으로 무엇을 할까요?" / header: "다음 단계"
- 선택지:
  - **대상 스냅샷 만들기** — `preview_targets`로 조건별 인원을 보여주고 `create_target_snapshot`을 준비(실제 쓰기는 콘솔의 사람 승인 토큰 필요).
  - **실시간 동기화 설정** — 발송 결과 반영을 위해 `/crmforall-connector:sync`(또는 `sync_all`)를 안내.
  - **여기서 종료** — 온보딩 완료로 마무리하고 요약을 남긴다.

---

원칙 요약:
- `AskUserQuestion`은 위 **4개 지점에서만** 사용한다(과도한 질문 금지). 단순 진행은 그냥 수행.
- 각 단계 결과는 1~2줄 요약 + 다음 단계 안내.
- 비밀번호·`connectorSecret`·`cfit_`·전화번호 원문은 어떤 출력에도 넣지 않는다.
- 쓰기·발송 도구(`create_target_snapshot`·`send_kakao_message` 등)는 콘솔 승인 토큰이 있어야 동작한다.
